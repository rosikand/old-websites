Goal

The goal of my program is to create a mathematical search engine for math related problems. Time and time again, myself, and other students utilize search engines to find answers to mathematical expressions. Of course, I do not mean for cheating purposes, but rather for explanatory, learning purposes. However, entering in mathematical expressions using a keyboard can often be cumbersome difficult, and inefficientâ€”there is also no universal agreement as to what a user should enter for specific symbols. Thus, an easier, universal solution would be to allow the user to upload an image of their handwritten expression and use that as their search queryâ€”saving time for both the user and creating a universal search engine for all math text.

Summary of how the program works

In short, the pipeline of my program is as follows:

  1. User is asked to enter file names/paths of each particular digit/symbol in order, separated by spaces.
  1. Then, the next step is to convert these images into a useful format to serve as a query in the search engine. This is where machine learning comes into play. Each file is individually sent to a trained convolutional neural network to be classified.
  1. Once a file is a classified, the class name corresponds to a key stored in a dictionary. The value at that key, which is a specific string for that class, is retrieved. This is done for all image files.
  1. All class name values from the dictionary are taken and concatenated in sequential order.

How it was made

Preface and dataset

Classifying handwritten digits is a common task in machine learning. However, this was a difficult problem up until 2012 when it was discovered that convolutional neural networks (CNNs) work well for classifying images. From here, a deep learning/computer vision era was created. Due to this, in this past decade alone, we have seen things such as self-driving cars all through camera input. Thus, given the how powerful CNNs are, I utilized a convolutional neural network for this problem. Though classifying handwritten digits is a common task with a widely popular dataset (MNIST), there are not many people working on/very little datasets that include mathematical symbols in addition to the 0-9 digits. Luckily, I came across the dataset found here: https://www.kaggle.com/clarencezhao/handwritten-math-symbol-dataset.

In total, there are 16 different classes: 

MATH_SYMBOLS = {'decimal': '.', 'division': 'divide', 'eight': '8', 'equal': 'equals', 'five': '5', 'four': '4', 'minus': 'subtract', 'one': '1', 'plus': 'plus', 'seven': '7', 'six': '6', 'times': 'multiply', 'two': '2', 'zero': '0', 'three': '3', 'nine': '9'}

Building the convolutional neural network

To build the CNN, I utilized the powerful Keras deep learning library that sits on top of Tensorflow and Python. For the architecture, I used the VGG-19 architecture (see this paper: https://arxiv.org/abs/1409.1556), and tailored it to meet the needs of this project. Specifically, the standard output layer for the VGG-19 architecture is a 1000 neuron dense layer (since the architecture was built for the ImageNet competitionâ€”a competition for classifying 1000 classes). However, since the dataset I used only has 16 classes, the last layer was changed to a 16 neuron dense layer.
